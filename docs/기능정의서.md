# Edu-Sync AI 기능정의서

> 시스템 전체 기능을 역할별·모듈별로 상세 정의한 문서  
> 최종 업데이트: 2026-04-11

## 1. 수강생 기능

### 1-1. 로그인

| 기능 | 상세 설명 |
|------|-----------|
| **데모 로그인** | 로그인 화면에서 "수강생" 버튼 클릭 → 즉시 데모 계정(student_001)으로 접속, 별도 인증 없음 |
| **카카오 OAuth 로그인** | 카카오 인가 코드를 통해 로그인 → 최초 로그인 시 자동 회원가입 + 프로필(닉네임/사진) 저장 |
| **초대 코드 연결** | 멘토가 생성한 초대 링크(`/?invite=코드`)로 접속하면 해당 멘토에게 자동 배정 |
| **QR 코드 로그인** | PC에서 QR 생성 → 모바일에서 스캔 → "승인" 클릭 → PC에서 자동 로그인 (크로스 디바이스 세션 공유) |

### 1-2. AI 챗봇 (웹 — 카카오톡 스타일 UI)

| 기능 | 상세 설명 |
|------|-----------|
| **메시지 전송** | 하단 입력창에 텍스트 입력 → "전송" 버튼 클릭 → AI가 의도를 분류하여 적절한 에이전트가 응답 |
| **AI 의도 분류 (Router AI)** | 학생 메시지를 GPT-4o-mini가 분석하여 `agent_a`(행정·커리어), `agent_b`(학습·예약), `human_handoff`(멘토 상담) 중 하나로 자동 라우팅 |
| **Agent A 응답** | 행정, 취업, 큐레이션 관련 질문 → FAISS 벡터 검색(공통 지식 + 큐레이션 + 멘토 자료) → 검색 결과 기반 AI 답변 생성 |
| **Agent B 응답** | 학습 질문 답변 + 보충수업 예약/취소 플로우 처리 |
| **Human Handoff** | AI가 감정 상담·진로 고민 등 감지(confidence ≥ 0.8) → 멘토에게 1:1 상담 요청 자동 전달 → 학생에게 "멘토님께 전달했습니다" 안내 |
| **메시지 표시** | 봇 메시지: 왼쪽 흰색 말풍선 + 프로필 아이콘 + 봇 이름 + 시간 / 사용자 메시지: 오른쪽 노란색(#FEE500) 말풍선 + 시간 |
| **선택지 버튼** | AI가 제시한 선택지(Choice Buttons)를 pill형 버튼으로 표시 → 클릭하면 해당 텍스트로 메시지 전송 |
| **대화 이력 복원** | 로그인 시 이전 대화 기록을 DB에서 불러와 화면에 표시 |

### 1-3. 웰컴 메뉴 (4개 퀵 액션)

| 버튼 | 동작 |
|------|------|
| **📰 오늘의 큐레이션** | 오늘 날짜의 큐레이션 콘텐츠(채용정보/IT뉴스/AI타임스/자격증·공모전/개발트렌드)를 카드 형태로 표시. 없으면 "오늘 등록된 큐레이션이 없습니다" 안내 |
| **📅 조교 연결** | 보충수업 예약 플로우 시작 (→ 1-4 참조) |
| **📚 학습 팁** | 담당 멘토의 자료를 "최신 자료" / "기초 자료" 선택 → 해당 자료 카드 목록 표시 (제목, 요약, 원문 보기/다운로드) |
| **🙋‍♂️ 멘토 연결** | 1:1 멘토 상담 요청 → 멘토 대시보드에 알림 표시 |

### 1-4. 보충수업 예약 플로우

| 단계 | 상세 |
|------|------|
| **1단계: 메뉴 선택** | "예약하기" / "예약 취소하기" 버튼 표시 |
| **2단계: 날짜 선택** | 예약 가능한 날짜 목록 + 각 날짜의 가용 슬롯 수 표시 → 날짜 클릭 |
| **3단계: 시간 선택** | 선택한 날짜의 가용 시간대 목록 (조교명 + 시작~종료 시간) → 시간 클릭 |
| **4단계: 내용 입력** | "어떤 내용을 보충받고 싶으신지 간단히 적어 주세요" → 자유 텍스트 입력 |
| **5단계: 예약 완료** | AI가 입력 내용을 정규화(이름/전화/설명 정리) → 예약 확정 → 조교에게 AI 브리핑 리포트 자동 생성 |

### 1-5. 보충수업 예약 취소 플로우

| 단계 | 상세 |
|------|------|
| **1단계** | "예약 취소하기" 선택 |
| **2단계** | 학생의 현재 예약 목록 표시 (날짜/시간/조교명/내용 요약) |
| **3단계** | 취소할 예약 선택 → 예약 취소 처리 → 슬롯 상태가 "available"로 복원 |

### 1-6. 큐레이션 카드

### 1-8. 카카오톡 채널

| 기능 | 상세 |
|------|------|
| **카카오 웹훅 연동** | 카카오톡 채널로 메시지 전송 → `/api/kakao/webhook` → 웹 챗봇과 동일한 멀티 에이전트 AI가 응답 |
| **메뉴 키워드 매칭** | "기본상담/안녕/시작/메뉴" → 웰컴 메뉴, "큐레이션/뉴스/채용정보" → 오늘의 큐레이션, "조교 연결/보충수업/예약하기" → 예약 플로우, "학습팁/멘토 자료" → 학습 팁, "멘토 연결/1:1 상담" → 멘토 연결 |
| **응답 제한** | simpleText 990자 제한, 퀵 리플라이 최대 10개 |
| **타임아웃 처리** | 4.8초 타임아웃 → 초과 시 "죄송합니다, 응답 생성에 시간이 걸리고 있습니다" 안내 |

### 1-9. 1:1 멘토 상담 요청

| 기능 | 상세 |
|------|------|
| **요청 방법 1 — AI 자동 감지** | 학생이 슬럼프, 동기 부족, 진로 고민, 감정적 어려움 등을 언급 → Router AI가 confidence ≥ 0.8으로 `human_handoff` 분류 → 멘토에게 자동 알림 |
| **요청 방법 2 — 수동 요청** | 하단 "🙋‍♂️ 1:1 멘토 상담 연결" 버튼 클릭 또는 웰컴 메뉴에서 "멘토 연결" 선택 |
| **학생에게 표시** | "멘토님께 전달했습니다. 연락 드리겠습니다" + ⚠️ 아이콘 알림 |
| **멘토 대시보드** | "1:1 요청" 항목에 해당 학생 표시, 노란색 배지로 알림 |
| **멘토 수강생 관리** | 학생 목록에서 "1:1 요청" 배지 표시 → 학생 클릭하면 상세 타임라인 확인 가능 |
| **알림 해제** | 멘토가 수강생 관리에서 해당 학생의 handoff를 "해결" 처리 → 배지 제거 |

## 2. 멘토 기능

### 2-1. 로그인

| 기능 | 상세 |
|------|------|
| **데모 로그인** | "멘토" 버튼 클릭 → mentor_001 계정으로 즉시 접속 |
| **카카오 OAuth** | 수강생과 동일한 카카오 로그인 플로우 |

### 2-2. 멘토 대시보드

| 위젯 | 상세 |
|------|------|
| **오늘의 큐레이션** | 오늘 등록된 큐레이션 최대 3건 표시 (카테고리, 제목, 요약) |
| **최근 활동** | 담당 학생들의 최근 챗봇 대화 최대 20건 (학생명, 메시지 내용, 시간) |
| **조교 예약 현황** | 담당 학생들의 예약 현황 최대 8건 (학생명, 날짜, 시간, 내용 요약) |
| **최신 자료** | 내가 올린 최신 자료 최대 5건 (제목, 요약, 업로드일) |
| **기초 자료** | 내가 올린 기초 자료 최대 5건 |
| **1:1 상담 요청 (Pending Handoffs)** | AI가 감지하거나 학생이 직접 요청한 상담 대기 목록 → 학생명, 사유, 마지막 메시지 표시 |

### 2-3. 1:1 상담 요청 알림 및 해제

| 기능 | 상세 |
|------|------|
| **알림 표시** | 대시보드 "1:1 상담 요청" 위젯에 pending 상태의 요청 표시 |
| **학생 카드 배지** | 수강생 관리 화면에서 해당 학생 옆에 "1:1 요청" 노란색 배지 표시 |
| **알림 해제** | 수강생 관리 → 해당 학생 클릭 → handoff 해제(dismiss) → 배지 제거 + 상태 "resolved"로 변경 |

### 2-4. 수강생 관리

| 기능 | 상세 |
|------|------|
| **수강생 목록** | 담당 수강생 전체 표시 (이름 이니셜 배지, 희망 직무, 1:1 요청 배지) |
| **수강생 초대 링크** | "초대 링크 생성" 클릭 → 고유 코드 생성 → URL 복사 → 학생에게 공유하면 자동 연결 |
| **수강생 상세 — 조교 예약 내역** | 선택한 학생의 보충수업 예약 목록 (조교명, 날짜, 시간, 전화번호, 예약 내용 요약) |
| **수강생 상세 — 활동 타임라인** | 선택한 학생의 최근 30일 활동 이력 역순 표시: 챗봇 대화(chat), 검색 키워드(search), 큐레이션 조회(curation_view) |
| **타임라인 더 보기** | 기본 최근 이벤트 표시 → "더 보기" 클릭 시 이전 기록 로드 |

### 2-5. 지식 베이스 (자료 관리)

| 기능 | 상세 |
|------|------|
| **최신 자료 탭** | 멘토가 올린 최신 학습 자료 목록 (AI 생성 제목/요약, 업로드일, 14일 초과 시 stale 배지) |

| **링크 업로드** | 외부 URL 입력 → AI가 링크 내용 분석 → 제목·요약 자동 생성 |

## 3. 조교(TA) 기능

### 3-1. 로그인

| 기능 | 상세 |
|------|------|
| **데모 로그인** | "TA (조교)" 버튼 클릭 → ta_jung 계정으로 즉시 접속 |

### 3-2. 스케줄 관리 대시보드

| 기능 | 상세 |
|------|------|
| **월간 달력 뷰** | 해당 월의 모든 날짜를 격자로 표시, 각 날짜에 슬롯 상태를 색상으로 구분 |
| **슬롯 색상** | 초록: 예약 가능(available), 회색: 휴무/불가(blocked), 파란: 예약됨(booked) |
| **휴무일 표시** | 해당 날짜의 모든 시간이 blocked이면 "(휴무)" 표시 |
| **시간대 표시** | 각 날짜에 설정된 시간 범위 표시 (예: "14:00-17:00") |
| **월 이동** | ◀ / ▶ 버튼으로 이전/다음 달 이동 |

### 3-3. 스케줄 생성 방법

| 방법 | 상세 |
|------|------|
| **빠른 슬롯 생성** | 프리셋: 반복 요일 선택 + 시작/종료 시간 + 주 수(1~4주) → 1시간 단위로 자동 생성 |
| **일괄 슬롯 생성** | 시작일~종료일 + 반복 요일 → 해당 범위 내 모든 날짜에 1시간 단위 슬롯 생성 |
| **AI 스케줄 어시스턴트** | 자연어 입력 (예: "월금 14시~17시, 수요일 휴무") → AI(GPT-4o-mini)가 파싱 → 월간 스케줄 자동 생성 |
| **날짜별 개별 설정** | 미니 달력에서 특정 날짜 클릭 → 해당 날짜만 시간/모드(available/blocked) 개별 설정 → `date_override` 모드로 기존 스케줄에 영향 없이 적용 |

### 3-4. AI 스케줄 파싱

| 기능 | 상세 |
|------|------|
| **자연어 → 구조화** | "4월 14일부터 18일까지, 매일 14시~17시" → `{mode: "full", rules: [{weekdays: [0,1,2,3,4], start: "14:00", end: "17:00"}]}` |
| **모드 구분** | `full`: 월 전체 패턴 (요일 기반), `date_override`: 특정 날짜만 변경 |
| **미리보기** | AI가 생성한 계획을 텍스트로 요약 표시 → "적용" 클릭 시 실제 슬롯 생성 |
| **증분 적용** | `date_override` 모드는 해당 날짜만 삭제/재생성, 나머지 날짜 영향 없음 |

### 3-5. 예약 관리

| 기능 | 상세 |
|------|------|
| **예약 확인** | 달력에서 파란색으로 예약된 슬롯 확인 → 학생명(마스킹), 전화번호, 예약 내용 요약 표시 |

## 5. AI 에이전트 시스템

### 5-1. Router AI (의도 분류기)

| 항목 | 상세 |
|------|------|
| **입력** | 학생 메시지 텍스트 |
| **처리** | GPT-4o-mini가 메시지 분석 → 3가지 카테고리 중 하나로 분류 |
| **출력** | `{intent: "agent_a" | "agent_b" | "human_handoff", confidence: 0.0~1.0, reason: "분류 근거"}` |
| **agent_a 키워드** | 취업, 채용, 자격증, 공모전, IT뉴스, AI, 학원규정, 포트폴리오, 이력서, 면접, 인사말 등 |
| **agent_b 키워드** | 코딩, 파이썬, 자바, 클래스, 함수, SQL, HTML, CSS, JavaScript, React, 알고리즘, 에러, 디버깅, 예약, 조교 등 |
| **human_handoff 키워드** | 슬럼프, 동기부족, 감정적 어려움, 진로 고민, 자퇴 고려, 대인관계 등 |
| **confidence 기준** | human_handoff는 confidence ≥ 0.8 이상일 때만 실행 → 미달 시 agent_a로 폴백 (오탐 방지) |

### 5-2. Agent A (행정·커리어 에이전트)

| 항목 | 상세 |
|------|------|
| **역할** | 행정 질문, 커리어 안내, 큐레이션 전달, 멘토 자료 검색 |
| **검색 소스 1: 공통 지식** | FAISS 벡터 검색 (공통 지식 PDF, k=4) |
| **검색 소스 2: 큐레이션** | AI가 질문의 큐레이션 관련성 분석 → FAISS 의미 검색 (k=8) → 시간 범위 필터(오늘/이번주/지난주/최근4주/전체) |
| **검색 소스 3: 멘토 자료** | 학생의 담당 멘토 자료만 검색 (최신 k=3 + 기초 k=3) |
| **큐레이션 폴백** | 벡터 검색 결과 없으면 최근 3일 이내 자료 자동 표시 |
| **응답 형식** | JSON: 답변 텍스트 + 선택지 + 큐레이션 카드 + 멘토 자료 카드 |
| **이벤트 기록** | 큐레이션 반환 시 `curation_view` 이벤트 기록, 검색 키워드 `search` 이벤트 기록 |

### 5-3. Agent B (학습·스케줄 에이전트)

| 항목 | 상세 |
|------|------|
| **역할** | 학습 질문 답변, 보충수업 예약/취소, AI 브리핑 리포트 생성 |
| **예약 감지** | "예약", "보충수업", "조교 연결", "시간대", "취소" 키워드 감지 → 예약 플로우 시작 |
| **학습 질문** | 예약 키워드가 아니면 → GPT-4o-mini로 학습 안내 답변 생성 |
| **브리핑 리포트** | 예약 확정 시 학생의 최근 2주 검색 키워드 + 예약 내용을 AI가 분석 → 핵심 필요 사항 & 교수법 추천 자동 생성 |
| **예약 정규화** | 학생 입력(이름/전화/설명)을 AI가 조교가 읽기 좋은 형태로 정리 |

### 5-4. Human Handoff (멘토 연결 시스템)

| 항목 | 상세 |
|------|------|
| **트리거** | Router AI confidence ≥ 0.8 감정 감지, 또는 학생 수동 요청 |
| **큐 생성** | handoff_queue 테이블에 {student_id, reason, last_message, priority, status: "pending"} 저장 |
| **멘토 알림** | 멘토 대시보드 "1:1 상담 요청" 위젯에 표시, 수강생 관리 학생 카드에 "1:1 요청" 배지 |
| **해결 처리** | 멘토가 dismiss → status: "resolved", 배지 제거 |
