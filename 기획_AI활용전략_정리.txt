■ 설정한 사용자는 누구이며, 해결하려는 구체적인 문제점/불편함은 무엇인가요?

설정한 사용자는 4개 그룹으로 나누어 봤습니다.

1. 학원 수강생
기존 학원 공지, 채용 정보 등이 분산되어 있어서 놓치기 쉽습니다.
학원 수업 시간 특성상 09~22시까지 수업을 하는데, 22시에 수업이 끝나고 질문이 있거나 밤 늦게 필요한 질문이 생각날 때 멘토 선생님에게 연락하기 어렵습니다.
조교 시스템이 수동(카톡, 전화)으로 이루어져 있어 번거롭습니다.

2. 멘토
학생 개개인의 학습 이력, 활동을 추적하기 어렵습니다.
수십 명에게 동일 공지를 반복 전달해야 하는 불편함이 있습니다.
케어가 필요한 학생을 사전에 파악하기 힘듭니다.

3. 조교
학생이 무엇을 모르는지 파악이 어렵습니다.
스케줄 파악과 관리가 힘듭니다.

4. 관리자
채용 공고, 뉴스, 공모전 등 큐레이션 수집과 배포하는데 시간 소요가 큽니다.
학생들에게 정보가 전달되었는지 확인이 어렵습니다.


■ 문제를 해결하기 위한 솔루션의 핵심 기능은 무엇인가요?

AI 멀티 에이전트 아키텍처를 중심으로 핵심 기능을 설명드리겠습니다.

1. 멀티 에이전트 AI 챗봇 (Agent A, Agent B, Human Handoff)
학생이 챗봇에 메시지를 보내면, AI가 먼저 그 메시지의 의도를 파악합니다.
"행정 관련 질문인지, 학습 관련 질문인지, 아니면 사람과 직접 대화가 필요한 상황인지"를 자동으로 판단해서 적절한 담당 AI에게 넘겨줍니다.

Agent A는 행정, 커리어, 큐레이션 관련 질문을 담당합니다.
예를 들어 "요즘 채용 공고 뭐 있어?" 라고 물어보면, AI가 관리자가 올려둔 채용 정보 중 관련된 것을 찾아서 알려줍니다.
멘토가 올린 학습 자료도 여기서 검색해서 보여주고, 자료 원문 보기나 다운로드도 챗봇 안에서 바로 가능합니다.

Agent B는 학습 질문과 조교 보충 수업 예약을 담당합니다.
학생이 "보충수업 예약하고 싶어"라고 말하면, 예약 가능한 날짜를 보여주고, 시간을 고르고, 어떤 내용을 보충받고 싶은지 적으면 예약이 완료됩니다.
예약이 되면 AI가 그 학생의 학습 이력을 분석해서 "이 학생은 이런 부분이 약하고, 이런 걸 물어봤었다"는 브리핑 리포트를 자동으로 만들어서 조교에게 전달합니다.

Human Handoff는 AI가 처리하기 어려운 상담이 필요한 상황일 때 작동합니다.
예를 들어 학생이 진로 고민이나 감정적으로 힘든 이야기를 하면, AI가 이걸 감지해서 담당 멘토에게 바로 알림을 보냅니다.
멘토 대시보드에 "1:1 요청"이 표시되고, 멘토가 직접 학생과 상담할 수 있도록 연결해줍니다.

이 챗봇은 웹에서도 사용 가능하고, 카카오톡 채널과도 연결되어 있어서 학생들이 별도 앱 설치 없이 카카오톡에서 바로 질문하고 답변받을 수 있습니다.
카카오톡으로 질문해도 웹에서 질문해도 동일한 AI가 동일한 방식으로 응답합니다.

2. AI 큐레이션 자동 정리 시스템
관리자가 채용 공고 PDF나 뉴스 링크를 올리기만 하면, AI가 알아서 제목과 요약을 만들어줍니다.
관리자가 직접 정리할 필요 없이, 파일만 던져두면 AI가 "이 내용은 채용정보 카테고리고, 제목은 이거고, 요약은 이거다"라고 자동으로 정리합니다.
정리된 큐레이션 콘텐츠는 챗봇이 학생 질문에 맞춰서 자동으로 찾아서 보여줍니다.
학생이 "AI 관련 뉴스 뭐 있어?" 하면 관련 큐레이션을 검색해서 카드 형태로 보여주는 방식입니다.

3. 조교 스케줄 관리 + AI 브리핑
조교가 자연어로 "4월 14일부터 18일까지, 매일 14시~17시에 1시간 단위로 열어줘"라고 입력하면 스케줄이 자동으로 생성됩니다.
달력 화면에서 예약된 날짜는 색깔로 구분되고, 휴무일도 설정할 수 있습니다.
학생이 예약하면 AI 브리핑 리포트가 자동 생성되어서, 조교가 수업 전에 "이 학생은 뭘 모르고, 뭘 물어봤고, 뭘 보충해줘야 하는지" 미리 파악할 수 있습니다.

4. 멘토 자료 업로드 + 챗봇 자동 연동
멘토가 PDF나 파일을 대시보드에서 올리면, AI가 내용을 읽고 제목과 요약을 자동으로 만듭니다.
이 자료는 벡터 데이터베이스에 자동으로 저장되어서, 나중에 학생이 관련 질문을 하면 챗봇이 해당 멘토 자료를 찾아서 답변에 포함시킵니다.
학생은 챗봇 안에서 자료의 제목, 요약을 보고 원문 보기나 다운로드를 바로 할 수 있습니다.


■ 이 솔루션이 도입되었을 때 기대되는 개선점이 무엇인가요?

1. 수강생 측면
24시간 AI가 즉시 응답해서 대기 시간이 사라집니다. 밤 11시든 새벽 2시든 질문하면 바로 답변을 받을 수 있습니다.
채용, 뉴스, 공모전 등 정보가 챗봇 안에서 자동으로 노출되어서 정보를 놓칠 일이 줄어듭니다.
보충 수업 예약이 대화형으로 날짜 선택, 시간 선택, 내용 입력 3스텝 이내에 완료됩니다.
카카오톡에서 바로 이용가능해서 별도 앱 설치나 웹사이트 접속 없이도 편리하게 사용할 수 있습니다.

2. 멘토 측면
담당 학생들의 최근 질문 기록, 예약 알림, 활동 타임라인을 대시보드 한 화면에서 확인할 수 있습니다.
학생별로 어떤 질문을 했고, 어떤 자료를 받았는지 추적이 가능해서, 면담 시 "요즘 뭐 공부해?" 물어볼 필요 없이 데이터 기반으로 파악할 수 있습니다.
AI가 못하는 감정 상담이나 진로 상담 영역에서 선제적으로 케어가 가능합니다.

3. 조교 측면
자연어로 스케줄을 설정하니까 관리 시간이 크게 줄어듭니다.
학생이 예약하면 AI 브리핑이 자동으로 만들어져서, "이 학생이 뭘 모르는지" 미리 알고 수업 준비를 할 수 있습니다.

4. 운영자 측면
큐레이션 콘텐츠는 파일이나 링크만 올리면 AI가 정리하고 배포까지 해줍니다.
반복적인 정리 업무가 없어지고, 학생 참여 데이터가 자동으로 쌓입니다.


■ AI 활용 전략

이 프로젝트에서 사용할 AI 도구(Claude Code, Cursor, Codex 등)와 모델은 무엇이며, 선택한 이유는 무엇인가요?

이 프로젝트에서는 크게 3가지 AI 도구를 사용했습니다.

1. GitHub Copilot (Claude Opus 4)
프로젝트 전체 코드를 작성하고 디버깅하는 데 사용했습니다.
선택한 이유는, Agent 모드에서 프로젝트 폴더 전체를 컨텍스트로 잡고 작업할 수 있어서 파일 간 연결이 끊어지지 않고 일관된 코드를 만들 수 있기 때문입니다.
예를 들어 백엔드 API를 수정하면 프론트엔드에서 그 API를 호출하는 부분도 같이 수정해줍니다. 한 번에 여러 파일을 동시에 편집할 수 있다는 점이 큰 장점이었습니다.

2. OpenAI GPT-5.4-mini (프로덕트에 내장된 AI)
실제 서비스 안에서 동작하는 AI입니다. 챗봇 응답, 큐레이션 요약, 브리핑 리포트 생성 등에 사용됩니다.
선택한 이유는, 비용이 저렴하면서도 한국어 성능이 뛰어나기 때문입니다.
학원 서비스 특성상 모든 대화가 한국어로 이루어지는데, 한국어 이해도와 생성 품질이 높아야 학생들이 자연스럽게 사용할 수 있습니다.
비용도 중요한데, 학생 수십 명이 매일 질문을 하면 API 호출이 많아지기 때문에 mini 모델을 선택해서 비용을 낮췄습니다.

3. FAISS (벡터 검색 엔진)
지식 베이스, 큐레이션, 멘토 자료를 검색하는 데 사용합니다.
선택한 이유는, 별도의 데이터베이스 서버를 띄울 필요 없이 파일 형태로 동작하기 때문입니다.
학원 규모의 문서량(수백~수천 건)에서는 충분한 검색 성능을 보여주고, 서버 비용도 들지 않습니다.
벡터 검색이라는 게 쉽게 말하면, 학생이 "AI 뉴스 알려줘"라고 질문했을 때 단순히 "AI"라는 단어가 포함된 문서만 찾는 게 아니라, 의미적으로 비슷한 내용의 문서까지 찾아주는 기술입니다.


■ 각 AI 도구별 활용 전략을 작성해주세요.

1. GitHub Copilot 활용 전략

개발 과정에서 Copilot의 Agent 모드를 주로 사용했습니다.
Agent 모드란, 단순히 한 줄 코드 추천이 아니라 프로젝트 전체 구조를 파악한 상태에서 "이런 기능 만들어줘"라고 지시하면 관련 파일을 여러 개 동시에 수정해주는 방식입니다.

실제 활용 예시를 들면,
"학생 챗봇에서 보충수업 예약 기능 추가해줘"라고 지시하면 백엔드 API 엔드포인트 생성, 프론트엔드 UI 추가, 데이터베이스 모델 수정까지 한 번에 처리됩니다.
코드를 만든 후에는 자동으로 오류 검사를 하고, 문제가 있으면 스스로 수정하는 루프를 돕니다.

보안 관련해서는 API 키가 코드에 노출되지 않도록 환경변수로 분리하고, 사용자 입력 검증, 인증 토큰 확인 등 기본적인 보안 사항을 코드 생성 시 자동으로 적용하도록 했습니다.

2. GPT-5.4-mini 활용 전략 (서비스 내 AI)

이 프로젝트의 핵심인 멀티 에이전트 구조에 대해 설명드리겠습니다.

학생이 메시지를 보내면 가장 먼저 "Router AI"가 동작합니다.
Router AI는 그 메시지가 어떤 종류의 질문인지 분류하는 역할입니다.

분류 결과에 따라 3개의 담당 AI 중 하나로 넘어갑니다.
첫째, Agent A는 행정이나 커리어 관련 질문을 처리합니다. 학원 규정, 채용 정보, 뉴스 같은 것들을 벡터 검색으로 찾아서 답변합니다. 멘토가 올린 학습 자료도 여기서 검색됩니다.
둘째, Agent B는 학습 질문이나 조교 보충 수업 예약을 처리합니다. 예약 가능한 시간대를 실시간으로 확인해서 안내하고, 예약이 완료되면 AI 브리핑 리포트를 자동 생성합니다.
셋째, Human Handoff는 AI가 판단하기에 사람의 상담이 필요한 경우에 작동합니다. 멘토에게 알림을 보내서 직접 학생과 대화할 수 있게 연결합니다.

모든 AI 응답은 JSON 형식으로 고정해두었습니다.
쉽게 말하면, AI가 자유롭게 긴 글을 쓰는 게 아니라 정해진 틀에 맞춰서 답변하도록 만들었다는 뜻입니다.
이렇게 하면 AI가 엉뚱한 답변을 할 확률이 줄어들고, 프로그램이 AI 응답을 안정적으로 처리할 수 있습니다.

카카오톡 연동도 같은 AI 구조를 그대로 사용합니다.
카카오톡 채널로 학생이 메시지를 보내면, 웹 챗봇과 동일한 멀티 에이전트가 응답합니다.
학생 입장에서는 카카오톡 앱에서 대화하듯 자연스럽게 질문하면 되고, 뒤에서는 같은 AI가 같은 방식으로 동작합니다.
별도 앱 설치 없이 카카오톡만 있으면 이용 가능하다는 점이 학생들의 접근성을 크게 높여줍니다.

3. FAISS 벡터 검색 활용 전략

이 프로젝트에서는 3개의 독립된 벡터 저장소를 사용합니다.

첫 번째는 공통 지식 베이스입니다. 학원 규정, 교재 내용 같은 기본 자료가 저장되어 있습니다.
두 번째는 큐레이션 저장소입니다. 관리자가 올린 채용 공고, 뉴스, 공모전 정보가 저장됩니다.
세 번째는 멘토별 자료 저장소입니다. 각 멘토가 올린 자료가 멘토 단위로 분리되어 저장됩니다. 학생이 질문하면 자기 담당 멘토의 자료만 검색되도록 되어 있습니다.

검색 방식을 설명드리면, 학생이 질문을 하면 먼저 AI가 "이 질문의 의도가 뭔지, 어떤 카테고리인지, 최근 것만 원하는지" 등을 분석합니다.
그 다음에 벡터 검색으로 의미적으로 가장 유사한 문서를 찾아옵니다.
검색 결과가 없을 때를 대비해서, 최근 3일 이내 자료를 보여주는 방식도 준비해두었습니다.


■ 토큰 낭비를 최소화하고 유지보수성 및 재현성을 높이기 위한 전략이 있다면 작성해주세요.

토큰 절약, 유지보수, 재현성 3가지로 나눠서 설명드리겠습니다.

1. 토큰 절약
토큰이란 AI에게 질문하거나 답변받을 때 사용되는 일종의 단위인데, 많이 쓸수록 비용이 올라갑니다.

우선 GPT-5.4-mini 모델을 선택해서 기본 비용을 낮췄습니다.
AI 응답을 JSON 형식으로 고정해서 불필요하게 긴 답변을 만들지 않도록 했습니다.
벡터 검색 결과도 상위 4개만 가져오도록 제한해서, AI에게 전달하는 참고 자료 양을 조절했습니다.
큐레이션 원문도 400자까지만 잘라서 전달하고, 카카오톡 응답은 990자로 제한해서 과도한 텍스트 생성을 방지했습니다.
나중에 보안이 중요한 환경에서는 외부 API 대신 내부 서버에 설치하는 AI 모델로 교체하는 것도 가능하도록 설계했습니다.

2. 유지보수성
유지보수가 쉽다는 건, 나중에 수정하거나 기능을 추가할 때 코드를 많이 고치지 않아도 된다는 뜻입니다.

AI 모델을 바꾸고 싶을 때 환경변수 한 줄만 수정하면 OpenAI에서 다른 모델로 전환할 수 있습니다.
LLMProvider라는 중간 계층을 만들어서, AI 모델이 바뀌어도 나머지 코드는 수정할 필요가 없도록 했습니다.
에이전트별 AI 지시문(프롬프트)이 각각 별도 파일에 분리되어 있어서, Agent A의 동작 방식을 바꾸고 싶으면 해당 파일만 수정하면 됩니다.

3. 재현성
재현성이란, 누군가 이 프로젝트를 처음 받아서 실행했을 때 동일한 환경을 만들 수 있느냐의 문제입니다.

시드 데이터를 준비해서 서버를 처음 키면 데모용 계정과 샘플 데이터가 자동으로 생성됩니다.
환경변수 예시 파일을 제공해서, 어떤 설정값이 필요한지 바로 알 수 있습니다.
사용하는 라이브러리 버전을 고정해서, 시간이 지나도 같은 환경에서 동작하도록 했습니다.
Git 커밋 기록으로 기획부터 구현까지 전 과정을 추적할 수 있습니다.
